npm

  • https://www.npmjs.com/
  • https://docs.npmjs.com/

WHY

WHAT

  • https://www.npmjs.com/
  • node package manager
    • 以前的 npm 只是为了安装管理 node 包
    • 后来随着 npm 的社区越来越庞大，而且本身都是 JavaScript 的包，所以现在官方也不把自己叫 node package manager 了
    • 现在的官方称自己是 JavaScript package manager
  • npm 是一个包管理器，可以帮你解决下载第三方包的问题，依赖管理你的包的依赖问题

两层含义：

  • 网站
    • npm 本身是一个网站，所有你想要的比较流行的第三方包都存储在 npm 的服务器中
    • npm 本身就像 GitHub 一样，存储了很多第三方包，如果用户需要使用就直接下载即可
    • 在 npm 上所有的第三方包都由用户产生
  • 命令行工具

HOW

https://docs.npmjs.com/

安装

  1. 安装 node 环境
  2. node 环境自带了一个工具：npm
  3. 确保在控制台输入 npm --version 可以看到一个版本号就说明你的 npm 环境是 OK 的

基本使用

  • npm install 包名
    • 该命令会下载指定的包到当前目录，默认这样下载下来的包都会被放到 node_modules 目录中，默认安装下载目录不能修改
    • 可以在报名后面跟上 @版本号 下载指定的版本
  • npm uninstall 包名

package.json 包说明文件

package.json 是辅助 npm 使用的一个配置文件。该文件可以记录一下项目的基本信息，其中有一个最重要的属性：dependencies 用来保存项目的第三方依赖项。

该文件简单点可以手动新建（但是不推荐），我们可以使用 npm init 来辅助生成。

有了这个文件之后，别人拿到你的项目如果发现没有第三方包，就可以通过 npm install 一次性安装所有的依赖项。

所以建议每个项目都有一个 package.json 文件，用来保存项目的依赖项信息。

两个建议：

  1. 每个项目都要有 package.json 文件
  2. 安装包的时候都加上 --save 选项

全局包和本地包

在 npm 中，不仅发布了一些普通的 JavaScript 包，还有一个命令行工具包，例如：less、http-server、browser-sync、等等。。。

除了 jQuery、bootstrap 这些包之外，还有一些命令行工具，例如 less ，安装 下来之后就可以像使用 npm、git 一样来使用这个工具了。

如果我说要往项目中安装一个包，则执行 npm install --save 包名 命令。该命令的执行位置必须在项目的目录下执行。
如果我说要往全局安装一个工具包，则执行 npm install --global 包名 命令。该命令的执行位置随便。

使用 npm 推荐的方式

  1. npm init -y 生成 package.json 文件
  2. 然后需要安装在项目中使用的第三方包的时候，通过 npm install --save 包名
  3. 如果需要全局工具包，则必须 npm install --global 包名 来安装
  4. 如果需要卸载本地项目包，则通过 npm uninstall --save 包名
  5. 如果需要卸载全局工具包，则通过 npm uninstall --global 包名

注意：

  • 安装本地项目包，必须在项目目录下执行
  • 安装全局工具包，在哪儿执行都可以

提示：

在项目中，不一定是所有文件都需要被 git 管理，
那这个时候更建议的做法就是把项目中不需要被 git 管理的文件或目录配置到 .gitignore 文件中。

如果项目需要上传到远程仓库，则在 .gitignore 文件中把 node_modules 给屏蔽掉

问题

  • 淘宝镜像源
  • 下载全局命令行工具
    • 全局命令行工具其实就是一个软件，该软件提供了一个命令用来让你使用
    • 所以使用全局命令行工具的前提是你知道这个工具是干嘛的
    • 然后全局安装：npm install --global 包名
    • 只要是第三方包说明了它具有命令行工具的能力，则这个包就可以往全局装，装好之后就可以使用这个包提供的命令了
  • package-lock.json
  • npm 版本
    • 不同的 node 环境自带的 npm 版本是不一致的，node 版本越高，npm 版本也就越高
    • 可以通过 npm install --global npm 自己升级自己
  • 下载 browser-sync 不成功
      1. 没有权限，解决方案就是以管理员身份运行 cmd 执行下载
      1. 网络问题，最起码把镜像源切换到淘宝，然后多尝试几次
